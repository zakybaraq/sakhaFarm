<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Migration → Database | Migration SQL runs with full database privileges — must be reviewed before execution |
| Seed → Database | Seed data is trusted input — no user-provided values at this stage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Tampering | Migration files | mitigate | Migration files are auto-generated by Drizzle Kit from schema definitions — never hand-edit SQL. Review migration diff before running. Files committed to git for audit trail |
| T-01-10 | Tampering | Seed script | mitigate | Seed data uses hardcoded constants only — no user input, no environment-dependent values. Default roles marked is_default=true for application-level protection against deletion (Phase 3) |
| T-01-11 | Denial of Service | db:migrate | accept | Migration runs at deploy time with full DB access. Low-value target — attacker with migration access already has full DB control. Mitigated by restricting deploy pipeline access |
</threat_model>
