# Plan 02: Backend CRUD for Feed Types & Feed Brands

## Objective
Create full CRUD backend modules for feed-types and feed-brands following existing unit/plasma patterns.

## Tasks

### Task 1: Feed Types Backend Module
- Create `server/src/modules/feed-types/feed-types.service.ts` with:
  - `createFeedType(input, tenantId, userId)` — validates unique (tenantId, code), creates row
  - `listFeedTypes(tenantId)` — returns all active types for tenant
  - `getFeedType(id, tenantId)` — single type lookup
  - `updateFeedType(id, input, tenantId, userId)` — updates name/code, checks uniqueness
  - `toggleFeedType(id, tenantId, userId)` — flips is_active
  - `softDeleteFeedType(id, tenantId, userId)` — sets deletedAt, checks if referenced by feed_products
- Create `server/src/modules/feed-types/feed-types.controller.ts` with Elysia routes:
  - `GET /api/feed-types` → listFeedTypes
  - `POST /api/feed-types` → createFeedType
  - `GET /api/feed-types/:id` → getFeedType
  - `PUT /api/feed-types/:id` → updateFeedType
  - `PUT /api/feed-types/:id/toggle` → toggleFeedType
  - `DELETE /api/feed-types/:id` → softDeleteFeedType
- Create `server/src/modules/feed-types/feed-types.errors.ts` with:
  - `FeedTypeNotFoundError`, `DuplicateFeedTypeCodeError`, `FeedTypeInUseError`
- Register routes in `server/src/index.ts`

### Task 2: Feed Brands Backend Module
- Create `server/src/modules/feed-brands/feed-brands.service.ts` with:
  - Same CRUD pattern as feed-types, with additional `phone` field
- Create `server/src/modules/feed-brands/feed-brands.controller.ts` with same route pattern
- Create `server/src/modules/feed-brands/feed-brands.errors.ts`
- Register routes in `server/src/index.ts`

### Task 3: Update Feed Module for typeId/brandId
- Update `server/src/modules/feed/feed.controller.ts`:
  - Replace `phase` in create/update body with `typeId` and `brandId`
  - Add `typeId` and `brandId` to validation schema
- Update `server/src/modules/feed/feed.service.ts`:
  - Replace `phase` references with `typeId` and `brandId`
  - Update `getAllStock` to JOIN feed_types and feed_brands for typeName/brandName
  - Update `getStockForPlasmaFeed` similarly
  - Update `createSuratJalan` to work with new schema

## Verification
- `cd server && bun run typecheck` passes with zero errors
- All endpoints return correct responses with tenantId scoping
- Soft delete guard: deleting a feed_type referenced by feed_products returns error
