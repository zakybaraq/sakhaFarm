## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline gap between action icons |
| sm | 8px | Compact element spacing, gap between form fields inside modal |
| md | 16px | Default element spacing, table cell padding |
| lg | 24px | Page-level padding (`p: 3` = 24px), section separation |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Modal dialog: `maxWidth="sm"` fullWidth, `DialogContent` inner gap: 16px between fields (`gap: 2`)
- Table cell padding: 12px 16px (from MUI theme override — non-standard, already set globally)
- Action icon touch target: 32px (MUI `IconButton size="small"`)

Source: `client/src/pages/units/Units.tsx`, `client/src/pages/units/UnitModal.tsx`, `client/src/theme/index.ts`

### Delete Interaction

All three pages use `window.confirm()` inline (matching existing Units/Plasmas pattern):

```tsx
onClick={() => {
  if (confirm('Apakah Anda yakin ingin menghapus [entity] ini?')) {
    deleteMutation.mutate(row.original.id);
  }
}}
```

No separate confirmation Dialog component — consistent with existing pattern.

## Interaction States

### Toggle (isActive Switch)

- Checked (active): green `#2E7D32` — MUI primary color default on Switch
- Unchecked (inactive): grey — MUI default
- Optimistic update: switch flips immediately, reverts with error toast if API fails
- Error toast copy: `Gagal mengubah status: {error.message}`

### Row Actions

- Edit icon: default MUI `text.secondary` color (`#64748b` region)
- Delete icon: `color="error"` → `#D32F2F`
- Both `size="small"` `IconButton`, rendered in `Box` with `gap: 0.5` (4px), `justifyContent: center`

### Autocomplete fields (Feed Products modal — typeId, brandId)

- Options fetched via TanStack Query (`listFeedTypes`, `listFeedBrands`)
- Filter: only `isActive = 1` records shown in dropdown options
- Loading state: show `CircularProgress` inside Autocomplete `loading` prop
- No options text: "Tidak ada pilihan tersedia"

### Form Error Display

- Field-level: `helperText={errors.field?.message}`, `error={!!errors.field}`
- Form-level (API error): `Alert severity="error"` above first field inside `DialogContent`
