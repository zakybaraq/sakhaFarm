Create `server/src/db/seed/seed-default-roles.ts`:
- Define DEFAULT_ROLES constant as array of {name, description, isDefault: 1, tenantId: null, permissions: string[]}
- `seedDefaultRoles()` function:
  1. Call seedPermissions() first to ensure permissions exist
  2. For each default role: check if exists by name, if not → INSERT into roles
  3. After role exists, resolve permission names to IDs, INSERT into role_permissions with action='allow'
  4. Use transaction for each role+permissions to ensure atomicity
- Super Admin gets ALL permissions
- Admin Unit gets unit.*, plasma.*, cycle.*, recording.*, feed.*, inventory.read, audit.read
- Admin Plasma gets plasma.read, cycle.*, recording.*, feed.read, inventory.read
- Viewer gets *.read permissions only

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-12 | Elevation of Privilege | index.ts route ordering | mitigate | Register RBAC and user routes AFTER session/auth plugins so requirePermission guard has access to user context; verify plugin ordering in index.ts |

</threat_model>
