Main content area uses `ml` (margin-left) set to sidebar width on desktop, 0 on mobile.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Layout | User interactions (menu clicks, form submissions) |
| Login page → Auth API | Credentials transmitted over HTTPS to /api/auth/login |
| Route guard → Auth context | Client-side permission check for UI filtering only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Spoofing | Login form | mitigate | Server-side rate limiting on /api/auth/login (5 attempts/min); generic error messages; no client-side password storage per D-03 |
| T-08-07 | Tampering | Client-side route guard | accept | RequireAuth is UI-only filtering; backend enforces RBAC via requirePermission middleware per D-06 |
| T-08-08 | Information Disclosure | Breadcrumbs | accept | Breadcrumbs show route segments from URL only, no sensitive data |
| T-08-09 | Elevation of Privilege | Sidebar menu filtering | mitigate | Menu items hidden per user permissions from /auth/permissions; backend still validates all API calls server-side |
| T-08-10 | Denial of Service | Logout | accept | Logout is idempotent; repeated calls simply clear already-cleared state |
</threat_model>
