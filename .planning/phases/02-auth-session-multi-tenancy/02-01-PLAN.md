<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → /auth/login | Untrusted credentials (email/password) cross here |
| Client → Session cookie | httpOnly cookie sent to client, must be protected from XSS |
| /auth/register → Database | User input must be sanitized before DB insert |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | /auth/login | mitigate | Rate limit login to 5/min per IP via Redis counter; account lockout after 5 failed attempts |
| T-02-02 | Tampering | Session cookie | mitigate | httpOnly + Secure + SameSite=Strict cookie attributes; cookie signed by Lucia |
| T-02-03 | Information Disclosure | Password hash | mitigate | Password hash never exposed in API responses; getUserAttributes excludes passwordHash |
| T-02-04 | Denial of Service | /auth/register | mitigate | Rate limit registration to 10/min per IP; email uniqueness check before hash computation |
| T-02-05 | Elevation of Privilege | Session creation | mitigate | Session created only after successful password validation; no session without valid credentials |
</threat_model>
