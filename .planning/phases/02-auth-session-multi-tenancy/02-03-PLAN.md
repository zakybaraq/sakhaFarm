       - Export function `createRateLimiter(redisClient)` that returns Elysia plugin
       - Plugin provides `rateLimit(options)` beforeHandle guard with:
         - key: string (e.g., "login:{ip}", "api:{ip}", "heavy:{ip}")
         - window: number (seconds)
         - max: number (max requests in window)
       - Implementation:
         a. Extract IP from request: `context.request.headers.get("X-Forwarded-For") ?? context.request.headers.get("X-Real-IP") ?? "unknown"`
         b. Redis key: `ratelimit:{key}:{ip}`
         c. Use Redis INCR with EXPIRE (atomic): `const count = await redis.incr(redisKey); if (count === 1) await redis.expire(redisKey, window)`
         d. If count > max → return 429 "Too many requests. Try again in {remaining}s"
         e. Set response headers: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset
       - Pre-configured guards:
         - loginRateLimit: key="login", window=60, max=5 (per CONTEXT.md locked decisions)
         - apiRateLimit: key="api", window=60, max=100
         - heavyRateLimit: key="heavy", window=60, max=10
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IP address → Rate limiter | IP can be spoofed; rely on X-Forwarded-From from trusted proxy |
| Account lock → User access | Legitimate users may be locked out; need admin unlock path |
| Security headers → Browser | Headers must be set before any response body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Denial of Service | Rate limiter | mitigate | Redis-backed atomic INCR prevents bypass; per-IP keying; 5/min login limit |
| T-02-11 | Denial of Service | Account lockout | accept | 15-min auto-unlock; legitimate users can contact admin; low-impact DoS vector |
| T-02-12 | Tampering | Security headers | mitigate | Headers set via Elysia onAfterHandle hook (runs after route handler, before response sent) |
| T-02-13 | Spoofing | X-Forwarded-For IP | transfer | Trust proxy IP only from known load balancer; in dev mode use direct IP |
</threat_model>
