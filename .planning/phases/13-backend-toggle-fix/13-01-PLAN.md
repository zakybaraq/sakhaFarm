<objective>
Fix the backend to accept and process `isActive` toggle requests for units and plasmas.

Purpose: The PUT /:id body schema for both units and plasmas rejects `{ isActive }` with a 422 validation error (Elysia strips unknown fields), causing the frontend toggle to silently fail or receive a 500. This plan adds `isActive` to both PUT schemas and adds the corresponding service-layer handling that maps boolean `true/false` to int `1/0` for the `isActive` DB column.

Output: Two modified controller files (schema updated) and two modified service files (isActive handling added). No new endpoints or files created.
</objective>
