**Constants (at top of file, per D-04):**

```typescript
const VALID_DOC_TYPES = ['CP', 'Patriot', 'Ayam Unggul', 'MBU'] as const
```

**createCycle(input: { plasmaId: number, docType: string, chickInDate: string, initialPopulation: number }, tenantId: number, userId: string)**
- Validate DOC type: if !VALID_DOC_TYPES.includes(input.docType) → throw new Error(`Invalid DOC type. Must be one of: ${VALID_DOC_TYPES.join(', ')}`)
- Verify plasma belongs to tenant: query plasmas JOIN units WHERE plasmas.id = ? AND units.tenantId = ? AND isNull(plasmas.deletedAt).limit(1)
- If not found → throw CycleNotInTenantPlasmaError
- Capacity check: if input.initialPopulation > plasma.capacity → throw CycleCapacityExceededError(input.initialPopulation, plasma.capacity)
- Auto-calculate cycleNumber (per D-06): `db.select({ max: max(cycles.cycleNumber) }).from(cycles).where(eq(cycles.plasmaId, input.plasmaId))` → cycleNumber = (result[0]?.max ?? 0) + 1
- Insert: `db.insert(cycles).values({ plasmaId: input.plasmaId, cycleNumber, docType: input.docType, chickInDate: input.chickInDate, initialPopulation: input.initialPopulation, status: 'active', totalFeedKg: '0' })`
- Fire-and-forget audit with action: 'create' (Chick-In)
- Return created cycle

**listCycles(tenantId: number, plasmaId?: number, status?: string)**
- Query: `db.select({ ...cycles fields, plasmaName: plasmas.name, unitName: units.name }).from(cycles).leftJoin(plasmas, eq(cycles.plasmaId, plasmas.id)).leftJoin(units, eq(plasmas.unitId, units.id)).where(and(eq(units.tenantId, tenantId), isNull(cycles.deletedAt)))`
- If plasmaId provided: add `eq(cycles.plasmaId, plasmaId)`
- If status provided: add `eq(cycles.status, status)`
- Order by: `desc(cycles.chickInDate)`
- Return array

**getCycle(id: number, tenantId: number)**
- Query: same join pattern as listCycles, add `eq(cycles.id, id)`, limit(1)
- If result.length === 0 → throw CycleNotFoundError
- Return result[0]

**updateCycle(id: number, input: { docType?: string, chickInDate?: string, initialPopulation?: number }, tenantId: number, userId: string)**
- Get existing: same as getCycle
- If not found → throw CycleNotFoundError
- If status !== 'active' → throw InvalidCycleStatusTransitionError(existing.status, 'update')
- If input.docType → validate against VALID_DOC_TYPES
- If input.initialPopulation → capacity check against plasma.capacity
- Update: `db.update(cycles).set(input).where(eq(cycles.id, id))`
- Fire-and-forget audit with oldValue/newValue

**softDeleteCycle(id: number, tenantId: number, userId: string)**
- Get existing: same as getCycle
- If not found → throw CycleNotFoundError
- Check daily recordings: `db.select({ count: count() }).from(dailyRecordings).where(eq(dailyRecordings.cycleId, id))`
- If count > 0 → throw CycleHasRecordingsError(id, count)
- Update: `db.update(cycles).set({ deletedAt: new Date() }).where(eq(cycles.id, id))`
- Fire-and-forget audit with action: 'delete'

**completeCycle(id: number, input: { harvestDate: string, finalPopulation: number }, tenantId: number, userId: string)**
- Get existing: same as getCycle
- If not found → throw CycleNotFoundError
- If existing.status !== 'active' → throw InvalidCycleStatusTransitionError(existing.status, 'completed')
- Update: `db.update(cycles).set({ status: 'completed', harvestDate: input.harvestDate, finalPopulation: input.finalPopulation }).where(eq(cycles.id, id))`
- Fire-and-forget audit with action: 'complete', newValue: JSON.stringify({ status: 'completed', harvestDate, finalPopulation })

**failCycle(id: number, input: { harvestDate: string, notes?: string }, tenantId: number, userId: string)**
- Get existing: same as getCycle
- If not found → throw CycleNotFoundError
- If existing.status !== 'active' → throw InvalidCycleStatusTransitionError(existing.status, 'failed')
- Update: `db.update(cycles).set({ status: 'failed', harvestDate: input.harvestDate }).where(eq(cycles.id, id))`
- Fire-and-forget audit with action: 'fail', newValue: JSON.stringify({ status: 'failed', harvestDate, notes: input.notes ?? null })

Imports needed: { eq, and, isNull, desc, count, max } from 'drizzle-orm', { cycles } from '../../db/schema/cycles', { plasmas } from '../../db/schema/plasmas', { units } from '../../db/schema/units', { auditLogs } from '../../db/schema/audit_logs', { dailyRecordings } from '../../db/schema/daily_recordings', { db } from '../../config/database'

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted input (plasmaId, docType, chickInDate, initialPopulation, harvestDate) crosses here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Tampering | cycle.controller.ts body validation | mitigate | Elysia t.Object validation on all POST/PUT bodies — rejects malformed input before service layer |
| T-04-11 | Information Disclosure | listCycles/getCycle tenant scoping | mitigate | Every query joins cycles→plasmas→units and filters `eq(units.tenantId, tenantId)` — no cross-tenant data leakage |
| T-04-12 | Repudiation | audit_logs fire-and-forget | accept | Audit is best-effort; if audit fails, main operation still succeeds. Low risk — audit failure logged to server console |
| T-04-13 | Elevation of Privilege | RBAC permission guards | mitigate | requirePermission('cycle.*') on every route — unauthorized users get 403 before handler executes |
| T-04-14 | Spoofing | createCycle plasma ownership verification | mitigate | Service layer verifies plasma belongs to tenant (via unit chain) before inserting cycle — prevents assigning cycle to another tenant's plasma |
| T-04-15 | Tampering | Invalid status transitions | mitigate | completeCycle/failCycle check existing.status === 'active' before allowing transition — prevents reversing completed/failed cycles |
| T-04-16 | Denial of Service | Capacity check bypass | mitigate | createCycle validates initialPopulation <= plasma.capacity BEFORE insert — prevents overloading a plasma beyond its designed capacity |
</threat_model>
