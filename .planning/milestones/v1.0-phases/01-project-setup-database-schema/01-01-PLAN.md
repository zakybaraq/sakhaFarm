    Create `server/src/config/env.ts` with Zod schema that validates all required environment variables:

    ```typescript
    import { z } from 'zod'

    const envSchema = z.object({
      DATABASE_URL: z.string().url().startsWith('mysql://'),
      REDIS_URL: z.string().url().startsWith('redis://'),
      JWT_SECRET: z.string().min(32),
      SESSION_SECRET: z.string().min(32),
      PORT: z.coerce.number().int().min(1000).max(65535).default(3000),
      NODE_ENV: z.enum(['development', 'production', 'test']).default('development'),
    })

    export type Env = z.infer<typeof envSchema>

    export const env = envSchema.parse({
      DATABASE_URL: process.env.DATABASE_URL,
      REDIS_URL: process.env.REDIS_URL,
      JWT_SECRET: process.env.JWT_SECRET,
      SESSION_SECRET: process.env.SESSION_SECRET,
      PORT: process.env.PORT,
      NODE_ENV: process.env.NODE_ENV,
    })
    ```

    The parse call MUST throw a ZodError with a clear message if any required variable is missing or invalid. This ensures the server fails fast at startup with actionable error messages.

    import { env } from './env'

    export const redis = new Redis(env.REDIS_URL, {
      maxRetriesPerRequest: 3,
      retryStrategy: (times) => {
        if (times > 5) {
          console.error('[Redis] Max retries reached')
          return null
        }
        return Math.min(times * 200, 2000)
      },
    })

    redis.on('error', (err) => console.error('[Redis] Error:', err.message))

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment → Application | Untrusted env var values must be validated before use |
| Application → Database | Connection string contains credentials; pool config prevents injection |
| Application → Redis | Connection string contains credentials; retry strategy prevents infinite loops |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | env.ts | mitigate | Zod schema validates DATABASE_URL starts with mysql://, REDIS_URL starts with redis://, secrets min 32 chars — prevents malformed or malicious env values from reaching runtime |
| T-01-02 | Information Disclosure | .env files | mitigate | .env in .gitignore, .env.example template with placeholder values only — prevents credential leakage to version control |
| T-01-03 | Spoofing | database.ts pool | mitigate | Connection pool uses URI from validated env only; no hardcoded credentials; waitForConnections + queueLimit=0 prevents connection exhaustion attacks |
| T-01-04 | Denial of Service | redis.ts | mitigate | maxRetriesPerRequest=3 + exponential backoff retryStrategy prevents infinite retry loops that could exhaust resources |
</threat_model>
