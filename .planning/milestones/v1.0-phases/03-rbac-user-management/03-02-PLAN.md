  <behavior>
    - createUser: accepts {name, email, password (optional), roleId, tenantId}, auto-generates password if not provided, validates password complexity, checks email uniqueness, hashes password, inserts user, returns created user (without passwordHash)
    - listUsers: returns paginated list with joined role name and tenant name, supports ?page=N&limit=N
    - getUser: returns user by id with role name and tenant name → throws NotFoundError if missing
    - updateUser: updates name, roleId, tenantId — cannot update email or password through this endpoint → throws NotFoundError if missing
    - deactivateUser: sets isActive=0 → throws NotFoundError if missing
    - activateUser: sets isActive=1, isLocked=0 → throws NotFoundError if missing
    - resetPassword: generates new random password, hashes it, updates user, sets forcePasswordChange=1, returns new password (only time it's visible)
    - searchUsers: accepts query {name?, email?, roleId?, tenantId?, status?}, returns filtered users with pagination
    - importUsersFromCsv: accepts CSV string, parses "name,email,role,tenant" format, for each row: looks up role by name, looks up tenant by name, auto-generates password, creates user, returns {success: N, failed: N, errors: []}
  </behavior>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → User API | Untrusted input for user creation, CSV upload — must validate all fields |
| CSV Upload → Parser | Malicious CSV content could cause injection — parse safely, no eval |
| Password Reset → Session Store | Reset must invalidate all existing sessions for target user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | users.controller.ts → createUserHandler | mitigate | Validate email format with regex, enforce password complexity via validatePasswordComplexity, reject names with special characters |
| T-03-07 | Elevation of Privilege | users.routes.ts | mitigate | All user management routes guarded by requirePermission('user.*') — only superadmin or users with explicit permission can access |
| T-03-08 | Information Disclosure | listUsersHandler response | mitigate | Never return passwordHash in API responses; return only id, name, email, role, tenant, status, lastLoginAt |
| T-03-09 | Repudiation | users.service.ts mutations | mitigate | Every user mutation writes to audit_logs with userId, action ('user.create', 'user.update', 'user.reset_password'), oldValue, newValue |
| T-03-10 | Denial of Service | importUsersHandler (CSV bulk import) | mitigate | Limit CSV to max 1000 rows per import, reject files > 1MB, process row-by-row with individual error handling so one bad row doesn't block all |
</threat_model>
