  <action>
    1. Create server/src/modules/auth/profile.service.ts:
       - getProfile(db, userId): Promise<User> — fetches user by id, excludes passwordHash
       - changePassword(db, lucia, userId, currentPassword, newPassword): 
         a. Fetch user with passwordHash
         b. Verify currentPassword matches (oslo/password verify)
         c. If mismatch → throw "Current password is incorrect"
         d. Validate newPassword meets complexity rules (min 8, upper, lower, number, special)
         e. Hash newPassword with bcrypt cost=12
         f. Update userTable: set passwordHash = new hash, forcePasswordChange = 0
         g. Invalidate ALL sessions: `await lucia.invalidateUserSessions(userId)`
         h. Create new session for current request: `const session = await lucia.createSession(userId)`
         i. Return { session, user }
       - forcePasswordChange(db, userId, newPassword): superadmin-only, sets new password + forcePasswordChange = 0, invalidates all sessions
    
    2. Extend server/src/modules/auth/auth.controller.ts:
       - GET /auth/me: 
         a. Requires authenticated user (from sessionMiddleware)
         b. If !user → return 401
         c. Returns user profile (no passwordHash)
       - POST /auth/change-password:
         a. Requires authenticated user
         b. Accepts { currentPassword, newPassword }
         c. Calls changePassword service
         d. On success: sets new session cookie via lucia.createSessionCookie(session.id)
         e. Returns 200 + { message: "Password changed successfully" }
    
    3. Per D-08: Superadmin password reset (generates temp password + sets forcePasswordChange) — implement in this plan:
       - POST /auth/admin/reset-password:
         a. Requires authenticated superadmin user (check role via RBAC)
         b. Accepts { userId }
         c. Generates random 12-char temp password
         d. Hashes it, updates userTable
         e. Sets forcePasswordChange = 1
         f. Invalidates all sessions for target user
         g. Returns 200 + { temporaryPassword: "..." } (only returned once)
  </action>

  </verify>

    - New session cookie set after password change
    - POST /auth/admin/reset-password generates temp password for superadmin

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /auth/change-password → Session | Must verify current password before allowing change |
| /auth/admin/reset-password → User data | Only superadmin can reset; temp password exposed in response once |
| Session invalidation → Active sessions | All sessions destroyed, including current |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-14 | Elevation of Privilege | /auth/admin/reset-password | mitigate | RBAC guard: only Super Admin role can access; validate actor permissions before execution |
| T-02-15 | Information Disclosure | Temp password in response | mitigate | Returned only once in API response; not logged; forcePasswordChange requires immediate change |
| T-02-16 | Tampering | Password change | mitigate | Current password verification required; all sessions invalidated on change; audit log entry |
</threat_model>
