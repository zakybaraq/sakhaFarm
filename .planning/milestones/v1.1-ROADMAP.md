# Milestone v1.1: UI/UX Improvement

**Status:** ✅ SHIPPED 2026-04-20
**Phases:** 1–12 (v1.1 numbering)

## Overview

v1.1 transformed the frontend from a functional-but-rough MVP into a polished, modern, mobile-responsive application. The milestone replaced the outdated MUI DataGrid with TanStack Table v8 across all 9 pages, connected all remaining pages to real backend APIs, established ESLint/JSDoc code quality standards, added audit log filtering, and delivered 17 client integration tests via MSW infrastructure.

## Milestone Summary

**Key Decisions:**
- TanStack Table v8 chosen over AG Grid / Material Table (headless, mobile-responsive, already installed)
- Backward-compatible ColumnDef API preserved across DataGrid → TanStack migration
- Filter UI built inline in AuditLog.tsx (not extracted — audit-specific, premature abstraction)
- RBAC query condition changed from `!!user?.tenantId` to `!!user` (superadmin edge case)
- MSW for client tests (network-level mocking, no running server needed)
- Server smoke tests deferred (Drizzle mock failures + Elysia app not exported for testing)

**Issues Resolved:**
- MUI DataGrid mobile responsiveness (replaced with TanStack Table v8)
- 158 ESLint problems reduced to 0
- RBAC superadmin roles not loading (undefined tenantId)
- Audit log backend endpoint missing (implemented `/api/audit/logs`)

**Issues Deferred to v1.2:**
- Toggle 500 errors on Units/Plasmas (backend service bug)
- Cycle status toggle not enforcing read-only
- Phone column missing from Plasmas table
- Recordings dropdown empty, Deviasi BW calculation guard
- Feed dropdowns not connected to DB
- Performance/Stock Resume/Audit reports using dummy data
- RBAC tabs data integrity

**Technical Debt Incurred:**
- Extensive @mui/material usage (legacy exception — D-06 allows existing MUI to remain)
- 34 pre-existing server Drizzle mock failures (unresolved)
- Elysia app not exported before `.listen()` (needed for server test harness)
- Dark mode deferred since Phase 1
